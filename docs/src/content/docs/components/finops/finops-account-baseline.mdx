title: FinOps Account Hardening
description: Auto-applied SCP for the FinOps account

When `org.ous.sharedServices.accounts.finOps` is set, `ScpFinOpsAccountBaseline` is
auto-attached to the FinOps account. No extra config required.

The FinOps account exists for one purpose — hold CUR Parquet data, the Glue catalog,
and Athena queries. The SCP enforces that scope.

## What it denies

| Category | Examples |
|---|---|
| Compute / data services | EC2, RDS, SageMaker, Bedrock, ECS, EKS, EMR, Redshift, OpenSearch, Kinesis, MSK, Step Functions, AppRunner, Lightsail, WorkSpaces, AppStream, Code* |
| Network primitives | VPC / IGW / NAT / TGW / VPC peering / Client VPN creation |
| IAM identity creation | `CreateUser`, `CreateAccessKey`, `CreateLoginProfile` (humans use SSO) |
| Org integrity | `LeaveOrganization`, `CloseAccount`, disabling CloudTrail / GuardDuty / Macie / Config |

`AWSControlTowerExecution` is exempted everywhere so Control Tower keeps managing the account.

Glue and Athena run on AWS-managed infrastructure — no VPC needed. `ec2:CreateVpcEndpoint`
is NOT denied, so VPC endpoints to S3/Glue remain possible.

## Adding more deny statements

Use `scpStatements` on the FinOps account record. Additive only — you can't override a
baseline `Deny` with an `Allow`.

<DualCode>
  <Fragment slot="ts">
    ```ts
    sharedServices: {
      ouId: 'ou-xxxx',
      accounts: {
        finOps: {
          accountId: '123...',
          scpStatements: [
            new iam.PolicyStatement({
              effect: iam.Effect.DENY,
              actions: ['s3:PutBucketReplication'],
              resources: ['*'],
            }),
          ],
        },
      },
    }
    ```
  </Fragment>
  <Fragment slot="python">
    ```python
    shared_services=dlz.OrgOuSharedServices(
        ou_id="ou-xxxx",
        accounts=dlz.OrgOuSharedServicesAccounts(
            fin_ops=dlz.DLzFinOpsAccount(
                account_id="123...",
                scp_statements=[
                    iam.PolicyStatement(
                        effect=iam.Effect.DENY,
                        actions=["s3:PutBucketReplication"],
                        resources=["*"],
                    ),
                ],
            ),
        ),
    )
    ```
  </Fragment>
</DualCode>

To replace the default deny list entirely (e.g. allow Lambda for a small data-prep
workflow), use `ScpFinOpsAccountBaseline.statements({ deniedServices: [...] })`.
